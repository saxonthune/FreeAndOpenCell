# Research sessions

## Purpose

When design surfaces an open question that can't be answered from the code — "is there a library for this?", "does this idea have a name?", "what's the state of the art?" — the answer goes here. Each doc is one session: framing, findings, sources, and what (if anything) to adopt.

## What belongs here

- Library surveys with one-liners and links
- Academic / prior-art summaries
- Comparisons between two or more approaches we considered
- Reading notes that shaped a design decision

## What doesn't

- The decision itself (goes in the design doc it informs)
- Implementation notes (those live with the code, or in a post-implementation review)
- General reference material we expect to re-read (promote to codex or its own section)
